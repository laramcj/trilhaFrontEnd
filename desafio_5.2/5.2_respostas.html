Desafio 5.2

Respostas:

a)

  • I. Os componentes são os principais blocos de construção para aplicações angulares.
  • II. Cada componente consiste em:
    Um modelo HTML que declara o que é renderizado na página
    Uma classe Typescript que define o comportamento
    Um seletor CSS que define como o componente é usado em um modelo
    Opcionalmente, estilos CSS aplicados ao modelo
  • III. Para criar um component usando o Angular CLI devemos:
    - De uma janela do terminal, navegar para o diretório que contém a aplicação.
    - Rodar o ng generate component command, onde é o nome do seu novo componente.
    Por padrão, este comando cria o seguinte:
    - Uma pasta nomeada pelo componente;
    - Um arquivo do componente, .component.ts ;
    - Um arquivo template, .component.ts ;
    - Um arquivo CSS, .component.css ;
    - Um arquivo de especificação de teste, .component.spec.ts ;
    Onde é o nome do seu componente.
  • IV.a Reutilização de componentes é importante para evitar a duplicidade de código. Desta maneira se tem um ganho de tempo e um código mais limpo.
  • V. Cada componente requer um seletor CSS. Um seletor instrui o Angular a instanciar esse componente sempre que encontrar a tag correspondente no modelo HTML. Este seletor instrui o Angular a instanciar este componente sempre que a tag 'app-component-overview' aparecer em um template.
  • VI. Um template é um bloco de HTML que diz ao Angular como renderizar o componente em seu aplicativo. Para definir um template para seu componente há duas maneiras: referenciando um arquivo externo ou diretamente no componente.
    - Para definir um template como um arquivo externo, adicione uma propriedade templateUrl ao decorador @Component. @Component({ selector: 'app-component', templateUrl: './component.component.html' })
    - Para definir um template dentro do componente, adicione uma propriedade de template ao decorador @Component que contém o HTML que você deseja usar. @Component({ selector: 'app-component-overview', template: '

    Hello World!

    ' })
    - Se você quiser que seu template ocupe várias linhas, use crase (`). Por exemplo: @Component({ selector: 'app-component-overview', template: `

    Hello World!

    This template definition spans multiple lines.

    ` })
  • VII.Declare os estilos de componente usados para seu modelo de uma das duas maneiras: referenciando um arquivo externo ou diretamente no componente.
    - Para declarar os estilos de um componente em um arquivo separado, adicione uma propriedade styleUrls ao decorador @Component. @Component({ selector: 'app-component', templateUrl: './component.component.html', styleUrls: ['./component.component.css'] })
    - Para declarar os estilos dentro do componente, adicione uma propriedade styles ao decorador @Component que contém os estilos que deseja usar. @Component({ selector: 'app-component-overview', template: '

    Hello World!

    ', styles: ['h1 { font-weight: normal; }'] })
    A propriedade styles usa uma matriz de strings que contém as declarações de regras CSS.
